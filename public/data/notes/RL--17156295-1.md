



# Policy Gradient

在计算reward的时候, reward可能是一个人为设计不可微分的rule-based模型, 是不可微分的, 需要用 **log-derivative trick** 跳过对reward规则的微分

```
y = sample(πθ)
r = reward(x, y)
loss = -r
loss.backward()   ❌
```

具体地, 想要获得reward的期望对策略模型参数的梯度:
$$
\nabla_\theta\mathbb{E}_{y\sim\pi_\theta(y|x)}\big[r(y)\big]
$$
由于r是人定的规则, r对y是不可导的. 把期望展开为:
$$
\sum_y r(y)\cdot \nabla\pi_\theta(y|x)
$$

上面的展开式在y的情况很多甚至是连续的情况下是无法计算的, 而且由于y是从分布里采样的,采样操作不可导,因此引入**log-derivative trick**：
$$
\nabla\pi_\theta=\pi_\theta\nabla\log \pi_\theta
$$
代回展开的期望里得到：
$$
\sum_y \pi_\theta\cdot r(y)\cdot \nabla\log \pi_\theta
$$
上面的式子相当于是下面的期望
$$
\mathbb{E}_{y\sim\pi_\theta(y|x)}\big[r(y)\nabla_\theta\log\pi_\theta\big]
$$
由于$\pi_\theta$从reward里转移出来，因此就不需要对reward求导，而且上面的梯度期望也正好就是原reward的梯度的期望，可以用monte carlo采样计算梯度期望。
注意monte carlo计算期望时，采样概率$\pi_\theta$要stop gradient:
```python
# 1. sample（不参与梯度）
with torch.no_grad():
    y = sample_from(pi_theta)

# 2. forward again（参与梯度）
logp = log_prob(pi_theta, y)

# 3. reward（常数）
r = reward(y)

# 4. loss
loss = - r * logp
loss.backward()
```
由于采样过程已经间接实现了加权（概率高的会更多地被采样到，因此不需要额外乘概率了）。如果采样时从均匀分布采样，那就需要乘上$\pi_\theta$. 具体如下:
如果从$\pi_\theta$采样，有
$$
\frac{1}{N}\sum_i^N r\log\pi_\theta
$$
如果从另一个分布q采样
$$
\frac{1}{N}\sum_i^N \frac{\pi_\theta}{q} r\log\pi_\theta
$$
当q是均匀分布式时，其q是离散的，也是N个可能时，有
$$
\sum_i^N \pi_\theta \cdot r\log\pi_\theta
$$
每种可能乘上其概率$\pi_\theta$.
这也是为什么PPO里会有一个ratio，因为采样的时候用的是旧的分布$\pi_{old}$. 而且PPO里用clip也是因为当$\pi_\theta$与$\pi_{old}$差距过大时，ratio会爆炸，导致训练崩溃。



# GRPO

https://chatgpt.com/c/69c64e0e-17c4-83a9-b205-c62d064ec258

group relative policy optimization

采样top k组输出，计算总的reward，对reward做归一化，从而简化reward的设置，规避了给reward的定scale range的问题。归一化reward后输出只有相对的好坏之分，不需要复杂的reward设计，相当于排序。

**计算步骤：**

**Step 1** 对同一个输入 $x$，从旧策略 $\pi_{old}$ 采样 $G$ 个输出 $\{y_1, \dots, y_G\}$

**Step 2** 计算每个输出的 reward，组内归一化得到 advantage：

$$
\hat{A}_i = \frac{r_i - \text{mean}(\{r_j\})}{\text{std}(\{r_j\})}
$$

**Step 3** 用 clipped surrogate objective 更新策略，同时加 KL 惩罚防止偏离参考模型：

$$
\mathcal{L}_\text{GRPO} = -\mathbb{E}_i\left[\min\!\left(\frac{\pi_\theta(y_i|x)}{\pi_\text{old}(y_i|x)}\hat{A}_i,\ \text{clip}(\cdot, 1{-}\varepsilon, 1{+}\varepsilon)\hat{A}_i\right)\right] + \beta\,D_\text{KL}(\pi_\theta \| \pi_\text{ref})
$$

无需 value network（critic），advantage 完全来自组内相对排名。

# PPO

Proximal Policy Optimization

用clip限制importance sampling里的ratio，避免ratio爆炸导致训练崩溃.

**计算步骤：**

Critic $V_\phi(s)$ 是一个独立的神经网络，输入状态 $s$，输出对当前策略下从该状态出发的期望累积 reward 的估计（即状态价值函数）。它不参与动作选择，只用于计算 advantage，减小梯度方差。

**Step 1** 用旧策略 $\pi_\text{old}$ 采集轨迹，用 critic $V_\phi$ 估计 advantage（通常用 GAE）：

$$
\hat{A}_t = \sum_{l=0}^{T}(\gamma\lambda)^l\delta_{t+l}, \quad \delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)
$$

**Step 2** 计算 importance sampling ratio：

$$
\rho_t = \frac{\pi_\theta(a_t|s_t)}{\pi_\text{old}(a_t|s_t)}
$$

**Step 3** Clipped surrogate objective（actor loss）：

$$
\mathcal{L}_\text{CLIP} = -\mathbb{E}_t\!\left[\min\!\left(\rho_t\hat{A}_t,\ \text{clip}(\rho_t, 1{-}\varepsilon, 1{+}\varepsilon)\hat{A}_t\right)\right]
$$

**Step 4** 同时优化 value loss，总 loss：

$$
\mathcal{L} = \mathcal{L}_\text{CLIP} + c_1(V_\phi(s_t) - \hat{R}_t)^2 - c_2\mathcal{H}[\pi_\theta]
$$

其中 $\mathcal{H}$ 是熵正则项，防止策略过早收敛。

## Actor 与 Critic 一起训练吗？

是的，两者在每次迭代里**一起训练**，有两种常见架构：

### 共享主干（shared backbone）

```text
输入 s → 共享层 → { actor head → π(a|s)
                  { critic head → V(s)
```

一次 forward 同时算两个 head，总 loss 直接反传到共享层。参数效率高，LLM 的 RLHF 通常用这种（policy 和 value head 共享语言模型主干）。

### 独立网络（separate networks）

```text
输入 s → actor 网络  → π(a|s)
输入 s → critic 网络 → V(s)
```

两个网络各自独立更新，灵活但参数量翻倍。

### 每次迭代的训练流程

1. 用 $\pi_\text{old}$ 采集一批轨迹（stop gradient，不参与训练）
2. 对同一批数据，多次 mini-batch 更新，每次同时计算：
   - actor loss $\mathcal{L}_\text{CLIP}$（来自 policy gradient）
   - critic loss $(V_\phi(s) - \hat{R})^2$（来自 TD/MC target）
3. 合并成总 loss 一起 backward
